USING YOUR GITHUB REPO ON AWS
================================================================================

After your code is on GitHub, on AWS it's super simple:

  # Clone your repository
  git clone https://github.com/YOUR_USERNAME/18794-HW3-DeepLab.git
  cd 18794-HW3-DeepLab
  
  # Activate PyTorch
  source activate pytorch_p38
  
  # Install packages
  pip install -r requirements.txt
  
  # Download dataset
  mkdir -p datasets/data
  cd datasets/data
  wget http://host.robots.ox.ac.uk/pascal/VOC/voc2012/VOCtrainval_11-May-2012.tar
  tar -xf VOCtrainval_11-May-2012.tar
  mkdir -p VOC2012_train_val
  mv VOCdevkit/VOC2012 VOC2012_train_val/VOC2012_train_val
  
  # Train
  cd ~/18794-HW3-DeepLab
  python main.py --model deeplabv3plus_resnet50 --output_stride 16 --batch_size 8 --lr 0.01 --lr_policy step --step_size 1000 --total_itrs 5000 --gpu_id 0

================================================================================
TROUBLESHOOTING
================================================================================

Problem: "git: command not found"
Solution: Install Git from https://git-scm.com/download/win

Problem: "Permission denied"
Solution: Use Personal Access Token (not password) - see above

Problem: "Repository not found"
Solution: Create the repository on GitHub.com first

Problem: Files too large (>100MB)
Solution: Check .gitignore is working. Dataset should NOT be uploaded.
